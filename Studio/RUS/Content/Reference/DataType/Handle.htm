Handle
 

Handles are a way to represent arbitrary 32-bit or 64-bit pointer memory values in AutoLISP. Since AutoLISP does not support memory pointers as a native type, OpenDCL encodes handles as entity names when returning them to AutoLISP. These values are not really entity names, but they can be passed to OpenDCL functions that expect handles, and OpenDCL will interpret them correctly as memory pointers.
